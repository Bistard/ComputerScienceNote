# 发布和订阅 - Publish-Subscribe Pattern

> 发布订阅式和EDA关系非常密切，但是命名上会稍微不一样。

* 发布者（Publisher）—— 对应的就是EDA的event producer。
* 中间商（Broker）—— 对应的就是EDA的event manager。
* 订阅者（Subscriber）—— 对应的就是EDA的event consumer。
